# Contributing to Sentinel Enterprise Suite

Thank you for your interest in contributing to Sentinel Enterprise Suite! We welcome contributions from the community.

## Development Setup

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/thribhuvan003/sentinel.git
    cd sentinel
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    # or
    yarn install
    ```

3.  **Start the development server:**
    ```bash
    npm run dev
    # or
    yarn dev
    ```

## Code Style

-   We use **Prettier** for code formatting. Please ensure your code is formatted before submitting a PR.
-   We use **ESLint** for linting. Fix any linting errors before pushing.
-   Follow standard React and TypeScript best practices.

## Pull Request Process

1.  Fork the repository and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## Commit Messages

-   Use descriptive commit messages.
-   Start with a capitalized verb (e.g., "Add feature", "Fix bug").
-   Keep the first line under 50 characters if possible.
